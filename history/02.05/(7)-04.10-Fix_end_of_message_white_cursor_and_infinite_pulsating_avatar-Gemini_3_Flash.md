User request: "fix texting white box in the end of the message and pulsating avatar animation doesnt stop"

## Objective Reconstruction
The user requested to fix two UI artifacts after a message finishes generating: the white block cursor `█` remaining at the end of the text, and the assistant avatar continuing to pulsate indefinitely.

## Strategic Reasoning
We resolved this by:
1. Creating a 1.5s inactivity timeout that automatically marks the text stream typing as complete (`setHasFinishedTyping(true)`) if no new tokens are appended.
2. Restricting the inline typing cursor to only display if the message is actively generating (`isAILoading === true`).
