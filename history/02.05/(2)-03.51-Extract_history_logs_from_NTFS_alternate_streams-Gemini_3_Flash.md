User request: "why are history reports empty?"

## Objective Reconstruction
The user noticed that the history reports in `history/01.05/` appeared to be empty (0 bytes).

## Strategic Reasoning
- On Windows, filenames cannot contain colons (`:`). 
- When previous models created files with colons (e.g. `(2)-18:00-...md`), NTFS interpreted the part after the colon as an **Alternate Data Stream (ADS)**.
- The contents were successfully written into the ADS, but the primary file before the colon remained completely empty.
- We wrote a script using Node and PowerShell to extract the stream contents and rename the files to use a `.` or `-` separator.

## Detailed Blueprint
- **Created**: `scratch/recover_history.js`
  - A utility to find NTFS alternate data streams, extract their contents, and restore them as properly named files.
- **Removed**: Original empty ADS-bound files to keep history tidy.

## Operational Trace
- Successfully extracted 42 history reports from NTFS Alternate Data Streams and saved them with valid filenames (e.g., `(11)-19.30-...md`).

## Status Assessment
All history reports have been fully recovered and are no longer empty.
