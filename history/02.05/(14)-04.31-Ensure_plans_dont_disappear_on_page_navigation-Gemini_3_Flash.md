User request: "when i switch to other page and then come back to routines all plans dissapear fix"

## Objective Reconstruction
The user noticed that upon navigating away from the Routine page and coming back, any newly generated plans disappear. This was caused by Next.js App Router client caching / stale server component page responses.

## Strategic Reasoning
To resolve this permanently, we:
1. Marked the Routine page as `force-dynamic` so that Next.js always re-executes the page fresh instead of serving old static cache.
2. Added a mounting `useEffect` in the client component that queries the latest plans from Supabase via Server Action on page load. This guarantees the freshest plans list displays immediately without relying on the stale Router Cache.

## Detailed Blueprint
- **Modified**: `src/app/admin/bot/routine/page.tsx`
  - Added `export const dynamic = 'force-dynamic'`.
- **Modified**: `src/app/admin/bot/routine/RoutineClient.tsx`
  - Added on-mount `useEffect` to fetch fresh plans using the server action.
