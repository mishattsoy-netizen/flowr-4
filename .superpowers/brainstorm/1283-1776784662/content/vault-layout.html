A — Per-Provider Widgets (Grid)

One widget per provider (Gemini, Groq, OpenRouter…). Each widget is a self-contained list with inline add, reorder, show/hide, edit, and delete. Tabs or provider pills at the top to filter/switch. Mirrors the RouterManager pattern exactly.

B — Single Unified List (Grouped)

One flat list of all keys, visually grouped by provider with color-coded badges. Provider sections are collapsible. Row actions: show/hide, edit (inline), reorder (up/down arrows), delete. Add form at the top.

C — Grid of Provider Widgets

2-column grid (like RouterManager). Each cell is one provider. Dedicated "Add key" button per provider. Up/down reorder arrows, eye/edit/delete per row. No separate register form — adding lives inside each widget.
